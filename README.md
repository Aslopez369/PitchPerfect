# PitchPerfect · 音频变调控制器

> 一款运行在 Windows 上的桌面小工具，实时调节应用程序音频输出的音调高低（-12 key ~ +12 key），支持**全局模式**（整台电脑的声音一起变调）与**单应用模式**（只给某一个应用变调）。

---

## 一、它能做什么

- 🎚️ **实时音调调节**：范围 **-12 key ~ +12 key**，以半音（semitone）为单位逐级微调，拖动即生效，无需重启应用。
- 🌐 **全局模式**：把系统默认播放设备切到 VB-Cable，所有正在播放的声音（音乐、视频、游戏、语音）统一变调。
- 🎯 **单应用模式**：只针对选中的某一个应用程序变调，其他应用声音不受影响。
- 📋 **应用列表自动扫描**：一键列出当前正在播放音频 / 运行中的应用程序，带图标和进程名，方便精准选择。
- 📊 **状态可视**：底部状态栏实时显示 VB-Cable 是否就绪、当前输出设备、处理延迟（ms）。
- ♻️ **安全恢复**：停止变调后自动还原系统默认播放设备，不会留下"无声"或"回环"的烂摊子。

---

## 二、工作原理（为什么需要 VB-Cable）

Windows 的公开音频接口**不支持按进程单独捕获声音**，因此 PitchPerfect 借助虚拟音频线 **VB-Cable** 做中转：

```
目标应用 / 系统默认  ──▶  VB-Cable Input（虚拟扬声器）
                                │
                                ▼  被 PitchPerfect 捕获（CABLE Output）
                          ┌──────────────┐
                          │  SoundTouch  │  实时变调（不改速度）
                          └──────────────┘
                                │
                                ▼  输出到真实扬声器（你听得见的设备）
```

- **全局模式**：程序自动把系统默认渲染设备设为 `VB-Cable Input`，捕获 `CABLE Output` 变调后写回真实扬声器；停止后自动还原。
- **单应用模式**：不动系统默认设备，由你手动把"目标应用"的输出指定到 `VB-Cable Input`，程序捕获并变调后输出到真实扬声器，从而只影响这一个应用。

> ⚠️ **VB-Cable 是必需依赖**，未安装时程序会提示并禁用开始按钮。它是一款免费的虚拟音频驱动（VB-Audio Virtual Cable）。

## 五、使用说明

### 第一步：确认 VB-Cable 已就绪
启动程序后，观察**左下角状态灯**：
- 🟢 绿色 = VB-Cable 已检测到，可以开始变调。
- 🔴 红色 = 未检测到，请安装 / 重装 VB-Cable 后重启程序。

### 模式一：全局变调（全部声音）
1. 选中顶部 **「全局模式」**。
2. 拖动中央滑块设定音调（-12 ~ +12，0 为原声）。
3. 点击 **▶ 开始变调** —— 系统默认设备会切到 VB-Cable，所有声音开始变调。
4. 变调过程中可随时拖动滑块实时调整。
5. 点击 **■ 停止变调** —— 自动还原系统默认播放设备。

> 提示：开始变调前，请先确保正常播放设备（如"扬声器"）正在出声，这样效果最直接。

### 模式二：单应用变调（只改某一个应用）
1. 选中顶部 **「单应用模式」**，程序会自动扫描一次应用列表。
2. 若列表为空，点击 **🔄 刷新列表**（建议先让目标应用正在播放声音，再刷新）。
3. 在列表里找到目标应用，拖动它的音调滑块，并勾选 **「变调」**。
4. **关键手动步骤**：由于 Windows 不能按进程自动路由，你需要把该应用的输出设备手动指定到 VB-Cable：
   - 打开 Windows **设置 → 系统 → 声音 → 更多声音设置**（或直接右键任务栏喇叭 → 声音设置）。
   - 进入 **「应用音量和设备首选项」**（App volume and device preferences）。
   - 找到目标应用，把它的**输出**设备改为 **`CABLE Input (VB-Audio Virtual Cable)`**。
   - 该应用声音随即经 VB-Cable 进入 PitchPerfect 变调后输出到扬声器，其余应用不受影响。
5. 取消勾选「变调」即可停止对该应用的处理。

> 💡 单应用模式一次只处理一个应用；勾选新的应用会自动停用上一个。

## 六、常见问题（FAQ）

**Q：开始变调后没声音？**
- 确认正常播放设备正在出声；检查右下角状态灯是否绿色（VB-Cable 就绪）。
- 全局模式：停止后重新开始的"无声"问题已在最新版本修复，请确保使用的是最新构建。

**Q：单应用模式勾选了但声音没变调？**
- 多半是没做上面的"手动路由"步骤——目标应用的输出设备必须设为 `CABLE Input`，否则声音不经过 PitchPerfect。

**Q：为什么全局变调会影响所有声音？这是 Bug 吗？**
- 不是。全局模式的设计就是接管系统默认设备，对所有声音统一变调。只想改某一个应用请用单应用模式。

**Q：把真实扬声器改名成带 "CABLE" 的字会怎样？**
- 旧版本可能误判。最新版本已改用**设备 ID 精确匹配**排除 VB-Cable，改名不会误伤真实设备。

**Q：需要管理员权限吗？**
- 不需要，普通用户权限即可运行。

## 七、技术栈与项目结构

**技术栈**
- C# / WPF（.NET 8，Windows Desktop）
- [NAudio](https://github.com/naudio/NAudio) 2.2.1 —— Windows Core Audio 设备枚举与音频捕获/播放

```
├── ViewModels/
│   ├── MainViewModel.cs          # 主视图模型：模式、命令、状态
│   └── AudioSessionViewModel.cs  # 单应用会话项视图模型

│   ├── AudioProcessingService.cs # 核心：变调管线与设备路由（全局/单应用）

├── Models/                       # 数据模型
├── Converters/                   # XAML 值转换器（图标 / 可见性）
└── app.manifest                  # 清单（asInvoker，DPI 感知）
```
